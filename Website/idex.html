3Ô∏è‚É£ Operators (·ûÄ·û∂·ûö·ûî·üí·ûö·ûò·û∂·ûé·ûú·û∑·ûí·û∏)

Arithmetic: +, -, *, /, %, ++, --

Assignment: =, +=, -=, *=, /=, %=

Comparison: ==, ===, !=, >, <, >=, <=

Logical: &&, ||, !

üìö Documentation

JavaScript operators are symbols used to perform operations on operands. They are classified into several types:

  • Arithmetic Operators: Perform mathematical operations
  • Assignment Operators: Assign values to variables
  • Comparison Operators: Compare two values and return a boolean
  • Logical Operators: Combine multiple conditions 

// Arithmetic Operators
let a = 10, b = 3;
console.log(a + b);  // 13
console.log(a - b);  // 7
console.log(a * b);  // 30
console.log(a / b);  // 3.333...
console.log(a % b);  // 1 (remainder)
console.log(a++);    // 10 (post-increment)
console.log(++a);    // 12 (pre-increment)

// Assignment Operators
let x = 5;
x += 3;  // x = x + 3 ‚Üí 8
x -= 2;  // x = x - 2 ‚Üí 6
x *= 4;  // x = x * 4 ‚Üí 24
x /= 3;  // x = x / 3 ‚Üí 8

// Comparison Operators
console.log(5 == "5");   // true (loose equality)
console.log(5 === "5");  // false (strict equality)
console.log(5 != "5");   // false
console.log(5 !== "5");  // true
console.log(5 > 3);      // true
console.log(5 < 3);      // false

// Logical Operators
console.log(true && false);  // false
console.log(true || false);  // true
console.log(!true);          // false

4Ô∏è‚É£ Conditional Statements

üìö Documentation

Conditional statements control the flow of execution based on conditions:

  • if/else: Executes code based on a condition
  • switch: Executes different code blocks based on different cases
  • ternary operator: Shorthand for simple if/else statements 

// if/else statement
let age = 18;

    console.log("Adult");

    console.log("Child");

if(score >= 90){
    console.log("A");
} else if(score >= 80){
    console.log("B");
} else if(score >= 70){
    console.log("C");

    console.log("F");

let day = 3;
switch(day){
  case 1: 
    console.log("Monday"); 
    break;
  case 2: 
    console.log("Tuesday"); 
    break;
  case 3: 
    console.log("Wednesday"); 
    break;
  default: 
    console.log("Unknown day");
}

// ternary operator

let activity = isRaining ? "Stay inside" : "Go outside";

5Ô∏è‚É£ Loops (·ûö·ûÑ·üí·ûú·û∑·ûõ)

üìö Documentation

Loops execute a block of code multiple times:

  • for: Executes code a specific number of times
  • while: Executes code while a condition is true
  • do...while: Executes code at least once, then repeats while condition is true
  • for...of: Iterates over iterable objects (arrays, strings)
  • for...in: Iterates over object properties

// for loop
for(let i=0; i<5; i++){
    console.log(i); // 0, 1, 2, 3, 4
}

// while loop
let i = 0;
while(i < 5){
    console.log(i);
    i++;
}

// do...while loop
let j = 0;
do {
    console.log(j);
    j++;
} while(j < 5);

// for...of loop (arrays)
let fruits = ["apple", "banana", "orange"];
for(let fruit of fruits){
    console.log(fruit); // "apple", "banana", "orange"
}

// for...in loop (objects)
let person = {name: "John", age: 30, city: "New York"};
for(let key in person){
    console.log(key + ": " + person[key]); // "name: John", "age: 30", etc.
}

// Breaking and continuing loops
for(let i=0; i<10; i++){
    if(i === 5) break; // exits the loop
    if(i % 2 === 0) continue; // skips to next iteration
    console.log(i); // 1, 3
}

6Ô∏è‚É£ Functions (·ûò·ûª·ûÅ·ûÑ·û∂·ûö)

üìö Documentation

Functions are reusable blocks of code that perform specific tasks:

  • Function Declaration: function name() { }
  • Function Expression: const name = function() { }
  • Arrow Function: const name = () => { }
  • Parameters & Arguments: Values passed to functions
  • Return Statement: Returns a value from a function 

// Function Declaration
function hello(name){
    return "Hi " + name;
}

// Function Expression
const hi = function(name) {
    return "Hi " + name;
};

// Arrow Function
const greet = (name) => "Hi " + name;

// Function with multiple parameters
function add(a, b) {
    return a + b;
}

// Function with default parameters
function multiply(a, b = 1) {
    return a * b;
}

// Higher-order function (function that takes/returns functions)
function createMultiplier(multiplier) {
    return function(num) {
        return num * multiplier;
    };
}
const double = createMultiplier(2);
console.log(double(5)); // 10

// Immediately Invoked Function Expression (IIFE)
(function() {
    console.log("This runs immediately!");
})();

7Ô∏è‚É£ Objects & Arrays

üìö Documentation

Objects and arrays are fundamental data structures in JavaScript:

  • Objects: Store key-value pairs
  • Arrays: Store ordered collections of values
  • Methods: Functions that are properties of objects

// Spread operator

9Ô∏è‚É£ Events

üìö Documentation

Events are actions that occur in the browser, which JavaScript can respond to:

  • Event Types: click, mouseover, keydown, submit, load, etc.
  • Event Handling: addEventListener, inline event handlers
  • Event Object: Contains information about the event
  • Event Propagation: Bubbling and capturing phases 

// Adding event listeners
document.getElementById("btn").addEventListener("click", function(){
    alert("Clicked!");
});

// Event object
document.getElementById("btn").addEventListener("click", function(event){
    console.log(event.target); // The element that was clicked
    console.log(event.type);   // "click"
});

// Multiple events
let element = document.getElementById("myElement");
element.addEventListener("mouseover", function() {
    this.style.backgroundColor = "yellow";
});
element.addEventListener("mouseout", function() {
    this.style.backgroundColor = "";
});

// Keyboard events
document.addEventListener("keydown", function(event) {
    console.log("Key pressed: " + event.key);
});

// Form events
document.getElementById("myForm").addEventListener("submit", function(event) {
    event.preventDefault(); // Prevent form submission
    console.log("Form submitted!");
});

// Event delegation
document.getElementById("parent").addEventListener("click", function(event) {
    if (event.target.classList.contains("child")) {
        console.log("Child clicked: " + event.target.textContent);
    }
});

üîü JSON Syntax

üìö Documentation

JSON (JavaScript Object Notation) is a lightweight data interchange format:

  • Syntax Rules: Uses key-value pairs, double quotes for strings
  • Data Types: String, number, boolean, null, object, array
  • Methods: JSON.stringify(), JSON.parse()
  • Usage: Data exchange between client and server 

// JavaScript object
let person = {
    name: "Sok",
    age: 20,
    hobbies: ["reading", "gaming"]
};

// Convert to JSON string
let jsonString = JSON.stringify(person);
console.log(jsonString);
// '{"name":"Sok","age":20,"hobbies":["reading","gaming"]}'

// Parse JSON string back to object
let parsedPerson = JSON.parse(jsonString);
console.log(parsedPerson.name); // "Sok"

// JSON with nested objects
let company = {
    name: "Tech Corp",
    employees: [
        { name: "Alice", position: "Developer" },
        { name: "Bob", position: "Designer" }
    ]
};

let companyJSON = JSON.stringify(company);
console.log(companyJSON);

// Handling invalid JSON
try {
    let invalidJSON = '{"name": "John", "age": }';
    let parsed = JSON.parse(invalidJSON);
} catch (error) {
    console.log("Invalid JSON:", error.message);
}

// Pretty printing JSON
let prettyJSON = JSON.stringify(person, null, 2);
console.log(prettyJSON);

1Ô∏è‚É£1Ô∏è‚É£ Classes & OOP (ES6)

üìö Documentation

Classes in JavaScript provide a cleaner way to work with objects and inheritance:

  • Class Declaration: class ClassName { }
  • Constructor: Special method for initializing objects
  • Methods: Functions defined inside a class
  • Inheritance: extends keyword for creating subclasses
  • Encapsulation: Private fields using # prefix 

// Basic class
class Car{
   constructor(name, year){
      this.name = name;
      this.year = year;
   }
   
   drive(){
      console.log(this.name + " is driving");
   }
   
   getAge(){
      let currentYear = new Date().getFullYear();
      return currentYear - this.year;
   }
}

let myCar = new Car("Toyota", 2018);
myCar.drive(); // "Toyota is driving"
console.log(myCar.getAge()); // Current year - 2018

// Class inheritance
class ElectricCar extends Car {
   constructor(name, year, battery){
      super(name, year); // Call parent constructor
      this.battery = battery;
   }
   
   charge(){
      console.log(this.name + " is charging");
   }
   
   // Override parent method
   drive(){
      console.log(this.name + " is driving silently");
   }
}

let myElectricCar = new ElectricCar("Tesla", 2020, "100kWh");
myElectricCar.drive(); // "Tesla is driving silently"
myElectricCar.charge(); // "Tesla is charging"

// Static methods
class MathHelper {
   static add(a, b) {
      return a + b;
   }
   
   static multiply(a, b) {
      return a * b;
   }
}

console.log(MathHelper.add(5, 3)); // 8
console.log(MathHelper.multiply(5, 3)); // 15

// Private fields (ES2022)
class BankAccount {
   #balance = 0; // Private field
   
   deposit(amount) {
      this.#balance += amount;
   }
   
   getBalance() {
      return this.#balance;
   }
}

let account = new BankAccount();
account.deposit(100);
console.log(account.getBalance()); // 100
// console.log(account.#balance); // Error: Private field

1Ô∏è‚É£2Ô∏è‚É£ Modules (import/export)

üìö Documentation

JavaScript modules allow you to split code into multiple files:

  • Export: Makes functions, classes, or variables available to other modules
  • Import: Brings exported values from other modules
  • Default Export: One default export per module
  • Named Export: Multiple named exports per module 

// file1.js - Named exports
export const x = 10;
export const y = 20;
export function add(a, b) {
    return a + b;
}
export class Calculator {
    multiply(a, b) {
        return a * b;
    }
}

// Alternative: Export at the end
const a = 5;
const b = 15;
function subtract(a, b) { return a - b; }
export { a, b, subtract };

// file2.js - Importing named exports
import { x, y, add, Calculator } from './file1.js';
console.log(x); // 10
console.log(add(5, 3)); // 8

// Import with aliases
import { x as first, y as second } from './file1.js';

// Import all as an object
import * as math from './file1.js';
console.log(math.x); // 10

// file3.js - Default export
const greeting = "Hello World!";
export default greeting;

// file4.js - Importing default export
import greeting from './file3.js';
console.log(greeting); // "Hello World!"

// Mixed imports
import greeting, { x, y } from './file1.js';

// Dynamic imports (load modules on demand)
async function loadModule() {
    const module = await import('./file1.js');
    console.log(module.x); // 10
}

1Ô∏è‚É£3Ô∏è‚É£ Promise & Async/Await

üìö Documentation

Promises and async/await handle asynchronous operations in JavaScript:

  • Promise: Object representing eventual completion/failure of async operation
  • Async Functions: Functions that return promises
  • Await: Pauses execution until promise is resolved
  • Error Handling: try/catch with async/await, .catch() with promises 

// Creating a Promise
const myPromise = new Promise((resolve, reject) => {
    // Asynchronous operation
    setTimeout(() => {
        const success = true; // Simulate success/failure
        if (success) {
            resolve("Data loaded successfully!");
        } else {
            reject("Error loading data!");
        }
    }, 2000);
});

// Using a Promise
myPromise
    .then(result => {
        console.log(result); // "Data loaded successfully!"
    })
    .catch(error => {
        console.error(error); // "Error loading data!"
    });

// Async/Await
async function getData(){
    try {
        let r = await fetch("https://jsonplaceholder.typicode.com/users/1");
        let data = await r.json();
        console.log(data);
        return data;
    } catch (error) {
        console.error("Error:", error);
    }
}

// Multiple promises with Promise.all
async function fetchMultipleUsers() {
    try {
        const [user1, user2] = await Promise.all([
            fetch("https://jsonplaceholder.typicode.com/users/1").then(r => r.json()),
            fetch("https://jsonplaceholder.typicode.com/users/2").then(r => r.json())
        ]);
        console.log("User 1:", user1);
        console.log("User 2:", user2);
    } catch (error) {
        console.error("Error fetching users:", error);
    }
}

// Promise chaining
fetch("https://jsonplaceholder.typicode.com/users/1")
    .then(response => response.json())
    .then(user => {
        console.log(user);
        return fetch(`https://jsonplaceholder.typicode.com/posts?userId=${user.id}`);
    })
    .then(response => response.json())
    .then(posts => {
        console.log(posts);
    })
    .catch(error => {
        console.error("Error in promise chain:", error);
    });
